01 Démarrer le TP dans JSLinux
1
Ouvrez JSLinux (attendez 15–20 sec.) :
https://bellard.org/jslinux/vm.html?url=alpine-x86.cfg&mem=192
2
Téléchargez tp_infiltration.sh sur votre ordinateur :
https://github.com/babash/TP-commandes-hacker/raw/main/tp_infiltration.sh
3
Importez — cliquez sur la flèche ↑ sous le terminal :
JSLinux — Alpine Linux
~ # ▌
© Fabrice Bellard↑  Cliquez ici — importer un fichier
4
Lancez :
~# . ~/tp_infiltration.sh # Le point . charge le script — NE PAS écrire "source"
⚠ Utilisez bien le point . — pas le mot source
5
Affichez les missions :
~# MISSION
02 Copier-coller dans JSLinux
Ctrl+V ne fonctionne pas. Coller : clic droit → Paste.
03 Vos 3 commandes
MISSIONAffiche les 15 missions avec contexte et statut [OK] ou [ ]
Q1 … Q15Succès → badge + score.   Échec → indice niveau 1. Retapez pour le niveau 2, puis 3 (solution).
STATUTScore rapide — liste des missions validées
04 — Indices progressifs (si bloqué)

Tapez Qn pour vérifier. Si la mission échoue, un indice s'affiche. Retapez pour en avoir un autre :

1er appelObjectif + commande
2e appelStructure avec ???
3e appelSolution complète
05 Les 15 missions
Phase 1 — Reconnaissance
Q1pwd
Q2ls
Q3cd
Q4cat
Phase 2 — Manipulation de fichiers
Q5touch [auto]
Q6cp [auto]
Q7mv [auto]
Q8mkdir [auto]
Phase 3 — Droits  ·  Phase 4 — Processus
Q9chmod 600 [auto]
Q10chmod u+x [auto]
Q11ps + kill
Phase 5 — Options et pipe |
Q12ls --help
Q13ls -la
Q14cat | grep
Q15history | grep
[auto] = le daemon valide automatiquement dès que la tâche est faite.  Question = Qn pose une question sur le résultat.
06 Réinitialiser (progression conservée)
~# . ~/tp_infiltration.sh # La progression est sauvegardée et restaurée automatiquement
Pour repartir de zéro : rm -rf ~/.mirage_save && . ~/tp_infiltration.sh
