01 Démarrer le TP dans JSLinux
1
Ouvrez JSLinux et attendez 15–20 secondes :
https://bellard.org/jslinux/vm.html?url=alpine-x86.cfg&mem=192
2
Téléchargez le fichier tp_infiltration.sh sur votre ordinateur :
https://github.com/babash/TP-commandes-hacker/raw/main/tp_infiltration.sh
Si le navigateur affiche le contenu → clic droit sur le lien → "Enregistrer le lien sous..."
3
Importez dans JSLinux — cliquez sur la flèche ↑ sous le terminal :
JSLinux — Alpine Linux x86
Welcome to Alpine Linux 3.x
~ # ▌
© Fabrice Bellard ↑  Cliquez ici — importer un fichier
Sélectionnez tp_infiltration.sh. Il est copié dans ~/.
4
Lancez le TP :
~# . ~/tp_infiltration.sh # Le point . charge le script — NE PAS écrire "source", non reconnu ici
⚠ Utilisez bien le point . et non le mot source
5
Affichez les 15 missions :
~# MISSION
02 Copier-coller dans JSLinux
Ctrl+V ne fonctionne pas. Pour coller : clic droit sur le terminal → Paste (ou menu navigateur → Coller).
03 Commandes disponibles
MISSIONAfficher les 15 missions avec leur statut [OK] ou [ ]
Q1 … Q15Vérifier une mission — indice précis si non validée
STATUTScore et progression globale
AGENT <n>Aide hors-ligne pour la mission n — ex : AGENT 7
Les missions marquées [auto] se valident seules dès que vous faites la bonne action.
04 — Aide hors-ligne : Agent ECHO

Bloqué ? Tapez AGENT suivi du numéro de la mission. ECHO analyse l'état réel de votre serveur et vous guide en 3 niveaux :

1Rappel de l'objectif et de la commande
2Structure avec ??? à compléter
3Solution complète

ECHO vous rappellera à chaque fois : "Reviens me voir si tu as besoin de plus d'aide."

05 Les 15 missions — dans l'ordre que vous voulez
Q1pwd
Q2ls
Q3man
Q4ls -la
Q5cd
Q6cat
Q7cat | grep
Q8history | grep
Q9touch [auto]
Q10find
Q11mkdir [auto]
Q12chmod 600 [auto]
Q13chmod u+x [auto]
Q14top + ps
Q15kill + mv [auto]
Q7–Q8 = introduction du pipe |.   [auto] = validation automatique sans taper Qn.
06 Problème ? Tout réinitialiser
~# . ~/tp_infiltration.sh # Recrée tout : arborescence, espion, progression
