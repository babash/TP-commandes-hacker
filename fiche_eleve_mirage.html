01 Démarrer le TP dans JSLinux
1
Ouvrez JSLinux (attendez 15–20 sec.) :
https://bellard.org/jslinux/vm.html?url=alpine-x86.cfg&mem=192
2
Téléchargez tp_infiltration.sh sur votre ordinateur :
https://github.com/babash/TP-commandes-hacker/raw/main/tp_infiltration.sh
Clic droit → "Enregistrer le lien sous..." si le navigateur affiche le contenu.
3
Importez dans JSLinux — cliquez sur la flèche ↑ sous le terminal :
JSLinux — Alpine Linux
Welcome to Alpine Linux
~ # ▌
© Fabrice Bellard↑  Cliquez ici — importer un fichier
Sélectionnez tp_infiltration.sh. Copié dans ~/.
4
Lancez le TP :
~# . ~/tp_infiltration.sh # Le point . charge le script — NE PAS écrire "source" (non reconnu)
⚠ Utilisez le point ., pas le mot source
5
Affichez les missions :
~# MISSION
02 Copier-coller dans JSLinux
Ctrl+V ne fonctionne pas. Coller : clic droit sur le terminal → Paste.
03 Vos 3 commandes
MISSION Affiche les 15 missions avec leur contexte et leur statut [OK] ou [ ]
Q1 … Q15 Succès → badge + score mis à jour.
Échec → indice niveau 1. Retapez pour le niveau 2, puis 3 (solution).
STATUT Affiche votre score et la liste des missions validées
04 — Indices progressifs (si bloqué)

Tapez Qn pour vérifier une mission. Si elle échoue, un indice s'affiche. Retapez la même commande pour un indice de plus :

1er appelObjectif + commande concernée
2e appelStructure avec ??? à compléter
3e appelSolution complète

Exemple : tapez Q7 trois fois pour aller jusqu'à la solution de la mission 7.

05 Les 15 missions
Q1pwd
Q2ls
Q3ls --help
Q4ls -la
Q5cd
Q6cat
Q7cat | grep
Q8history | grep
Q9touch [auto]
Q10find
Q11mkdir [auto]
Q12chmod 600 [auto]
Q13chmod u+x [auto]
Q14ps | grep
Q15kill+mv [auto]
[auto] = validé automatiquement par le daemon dès que la tâche est faite (sans taper Qn).  Q7–Q8 = introduction du pipe |.
06 Problème ? Tout réinitialiser
