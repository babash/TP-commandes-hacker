15 questions progressives · Pipe introduit en contexte · Agent ECHO (IA) · Auto-corrigé
00 — Présentation

Ce TP place les élèves dans un scénario fictif d'infiltration d'un serveur. Les 15 questions sont progressives : les premières sont quasi-guidées pour mettre tout le monde en confiance, la difficulté monte doucement vers les droits et les processus.

Nouveauté v3 : le pipe | est introduit en contexte réel (Q7) avant d'être réutilisé (Q8). L'agent ECHO (IA) est disponible à tout moment via la commande AGENT.

⚠ Un rappel légal est affiché à chaque lancement. Le TP est 100 % local et sans risque réseau.

01 — Installation & lancement pour les élèves

Deux commandes suffisent. Les élèves les recopient depuis la fiche élève distribuée en début de séance :

$ wget -O ~/tp_infiltration.sh https://raw.githubusercontent.com/babash/TP-commandes-hacker/main/tp_infiltration.sh
$ source ~/tp_infiltration.sh

Le script crée l'arborescence dans ~/infiltration_mirage/, lance le processus espion et affiche l'introduction narrative.

Commandes disponibles une fois chargé :

CommandeEffet
MISSIONAffiche les 15 questions avec leurs énoncés complets
Q1 … Q15Vérifie la réponse + indice précis si échec + rappel AGENT
STATUTScore et progression globale
AGENTOuvre une ligne avec l'agent ECHO (IA) — aide contextuelle
⚙ Avant la séance
  • Vérifiez que wget, curl et python3 sont disponibles sur les postes élèves
  • L'agent ECHO nécessite une connexion internet et l'accès à api.anthropic.com — vérifiez que le pare-feu du lycée ne bloque pas ce domaine
  • Sans connexion, le TP fonctionne normalement — seul AGENT sera indisponible
  • Testez le lancement complet sur un poste élève avant la séance
02 — L'agent ECHO
Rôle pédagogique

La commande AGENT ouvre une ligne de communication avec ECHO, un assistant IA (Claude Haiku) dont le system prompt lui donne connaissance du TP, des 15 questions et de la progression de l'élève. Il aide sans donner directement la réponse, reste dans la narration espionnage, et répond en texte brut (lisible dans le terminal).

L'agent peut expliquer un concept (le pipe, les droits octaux, les signaux kill…), débloquer un élève hésitant, ou valider une compréhension.

Ce qu'ECHO sait
  • Le texte des 15 questions et les commandes attendues
  • Les questions déjà réussies et celles restantes (lu depuis .progression)
  • Le contexte narratif de l'opération MIRAGE

ECHO ne voit pas le terminal de l'élève — il ne peut pas diagnostiquer une erreur de frappe. Il raisonne à partir de ce que l'élève lui décrit.

03 — Questions & Corrigé complet
Q1

Orientation — Où êtes-vous ?

navigation

Afficher le chemin complet du répertoire courant.

// corrigé
pwd

Validation automatique — la commande affiche $PWD et la question est validée immédiatement. Mise en confiance garantie.

Q2

Exploration — Premiers pas

navigation

Lister le contenu de ~/infiltration_mirage/serveur/ sans options.

// corrigé
ls ~/infiltration_mirage/serveur/

La vérification confirme que le dossier est lisible et affiche les fichiers visibles. Elle signale l'existence de fichiers cachés, créant un appel à l'action pour Q4.

Q3

Documentation — Le manuel de ls

man

Consulter man ls pour trouver l'option qui affiche les fichiers cachés.

// corrigé
man ls
# Chercher avec /  → taper /-a puis Entrée
# L'option -a (--all) est la réponse
# Quitter avec q

Question déclarative — validée par simple appui sur Q3. L'occasion d'apprendre la navigation dans man (/motif pour chercher).

Q4

Infiltration — Fichiers cachés

navigationls -la

Lister serveur/ avec tous les détails et les fichiers cachés. Appliquer l'option trouvée en Q3.

// corrigé
ls -la ~/infiltration_mirage/serveur/

La vérification confirme la présence de .fichier_cache et en affiche le contenu comme récompense narrative (message d'ECHO).

Q5

Navigation — Pénétrez la base

cd

Se déplacer dans ~/infiltration_mirage/serveur/.

// corrigé
cd ~/infiltration_mirage/serveur/

Contrôle via $PWD. Erreur fréquente : confondre ls et cd. L'indice rappelle la distinction.

Q6

Lecture — Premier contact

cat

Afficher le contenu de message_secret.txt à la racine de ~/infiltration_mirage/.

// corrigé
cat ~/infiltration_mirage/message_secret.txt

Le fichier mentionne le journal access.log — lien narratif vers Q7.

Q7

Analyse — Introduction du pipe |

★ pipe |cat · grep

Filtrer le journal serveur/logs/access.log pour n'afficher que les lignes contenant Transfert.

// corrigé
cat ~/infiltration_mirage/serveur/logs/access.log | grep Transfert
// explication affichée à l'élève
Sans pipe : cat access.log  →  affiche TOUT le fichier
Avec pipe : cat ... | grep Transfert  →  affiche uniquement les lignes avec "Transfert"

# Le | prend la SORTIE de gauche et la donne en ENTRÉE à droite

Point clé : la vérification affiche elle-même l'explication du pipe, le résultat filtré et l'interprétation narrative ("quelqu'un a transféré rapport.txt vers une IP externe"). Question déclarative — validée par Q7.

Q8

Renseignement — Pipe sur l'historique

pipe |history · grep

Réutiliser le pipe pour chercher passwd dans l'historique des commandes.

// corrigé
history | grep passwd

Résultat probablement vide (normal). La vérification l'explique et donne un exemple de ce qu'on trouverait dans un vrai scénario. Question déclarative.

Q9

Présence — Empreinte agent

touch

Créer un fichier vide agent.log dans serveur/.

// corrigé
touch ~/infiltration_mirage/serveur/agent.log

Vérification par existence du fichier. Simple et rapide — permet de reprendre de l'élan avant les questions plus techniques.

Q10

Localisation — Rapport confidentiel

find

Trouver tous les fichiers nommés rapport.txt dans toute l'arborescence.

// corrigé
find ~/infiltration_mirage/ -name "rapport.txt"

Résultat attendu : serveur/confidentiel/rapport.txt. Insister sur la différence find (récursif) vs ls (un seul niveau).

Q11

Exfiltration — Créer + copier

mkdir · cp

Créer serveur/exfiltration/ puis y copier rapport.txt.

// corrigé
mkdir ~/infiltration_mirage/serveur/exfiltration
cp ~/infiltration_mirage/serveur/confidentiel/rapport.txt \
   ~/infiltration_mirage/serveur/exfiltration/

La vérification distingue les deux erreurs possibles : dossier absent, ou dossier présent mais fichier non copié. Indice spécifique à chaque cas.

Q12

Sécurisation — chmod 600

droits octaux

Appliquer les droits 600 sur exfiltration/rapport.txt.

// corrigé
chmod 600 ~/infiltration_mirage/serveur/exfiltration/rapport.txt
Décomposition : 600 = rw-------
  User  : 6 = r(4) + w(2) = lecture + écriture
  Group : 0 = aucun droit
  Others: 0 = aucun droit

Vérification via stat -c "%a". Bon moment pour rappeler la table r=4, w=2, x=1.

Q13

Préparation — chmod u+x

droits relatifs

Rendre effacer_traces.sh exécutable pour son propriétaire.

// corrigé
# Mode relatif (recommandé ici)
chmod u+x ~/infiltration_mirage/serveur/effacer_traces.sh

# Mode octal (accepté aussi : 744, 755, 700…)
chmod 744 ~/infiltration_mirage/serveur/effacer_traces.sh

La vérification contrôle uniquement le bit x du propriétaire. Elle affiche le rappel u=user, +=ajouter, x=exécution.

Q14

Surveillance — top + ps | grep

pipe |top · ps · grep

Lancer top pour observer les processus, repérer espion_mirage, puis trouver son PID avec ps aux | grep.

// corrigé
# Étape 1 : observer (quitter avec q)
top

# Étape 2 : trouver le PID
ps aux | grep espion_mirage
# ou plus propre :
pgrep -f espion_mirage

La vérification affiche directement le PID trouvé si le processus tourne. Signaler aux élèves que la ligne grep elle-même apparaît dans les résultats — c'est normal.

Q15

Conclusion — kill + mv

killmv

Tuer espion_mirage avec kill, puis renommer rapport.txt en notes_vacances.txt.

// corrigé
# Tuer le processus (PID obtenu en Q14)
kill <PID>
# Si résistance :
kill -9 <PID>

# Renommer
mv ~/infiltration_mirage/serveur/exfiltration/rapport.txt \
   ~/infiltration_mirage/serveur/exfiltration/notes_vacances.txt

La vérification teste les deux conditions indépendamment et donne un retour précis pour chacune. Si les deux sont remplies → message de fin de mission.

04 — Conseils pédagogiques
Déroulement recommandé
  • 5 min — introduction narrative orale + rappel légal + distribution de la fiche élève
  • 10 min — téléchargement et lancement (Q1 et Q2 validés en quelques minutes — mise en confiance)
  • 60–70 min — travail autonome ; encourager à lire les messages d'ECHO
  • 15 min — correction collective des moments-clés : pipe (Q7), droits octaux (Q12), kill (Q15)
Points d'attention par question
  • Q3 man : certains élèves ne savent pas naviguer dans man. Rappeler /mot pour chercher et q pour quitter.
  • Q5 cd : erreur classique — taper ls serveur/ au lieu de cd serveur/. Insister sur la différence voir / aller.
  • Q7 pipe : laisser les élèves lire l'explication intégrée dans le vérificateur avant d'intervenir. Le vérificateur est conçu pour être auto-suffisant.
  • Q10 find : rappeler que find cherche récursivement — ls non. Montrer la différence sur un exemple.
  • Q12 chmod : décomposer 600 au tableau (r=4, w=2, x=1) si les élèves bloquent.
  • Q14 ps : la ligne grep espion_mirage qui s'affiche dans les résultats perturbe souvent. Expliquer que grep se voit lui-même dans la liste des processus.
  • Q15 kill : différence entre kill (signal SIGTERM, poli) et kill -9 (SIGKILL, immédiat) — bonne opportunité pour parler des signaux Unix.
Réinitialiser le TP

En cas de problème ou pour repartir de zéro :

source ~/tp_infiltration.sh

L'arborescence est recréée, le processus espion relancé, la progression réinitialisée.
