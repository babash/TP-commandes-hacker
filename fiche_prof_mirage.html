00 — Présentation

TP de terminal Linux pour Terminale NSI sur JSLinux Alpine (Linux dans le navigateur). Scénario fictif d'infiltration. 15 missions indépendantes réalisables dans n'importe quel ordre.

Nouveautés v8.0 : daemon entièrement indépendant (script sur disque, plus de fonctions shell), indices progressifs intégrés directement dans Qn (3 niveaux, 1 par appel), interface réduite à 3 commandes seulement.

⚠ Rappel légal intégré au script. 100 % local, aucun réseau externe requis.

01 — Installation dans JSLinux

Procédure à distribuer via la fiche élève :

1. Ouvrir bellard.org/jslinux/vm.html?url=alpine-x86.cfg&mem=192

2. Télécharger tp_infiltration.sh depuis GitHub sur la machine locale

3. Cliquer sur la flèche ↑ sous le terminal JSLinux pour importer

4. Lancer avec le point (pas source) :

~# . ~/tp_infiltration.sh

⚠ Ctrl+V inopérant dans JSLinux — coller via clic droit → Paste.

Les 3 commandes disponibles :

CommandeRôle exact
MISSIONAffiche les 15 missions par phase avec statut [OK]/[ ], contexte narratif et tâche à accomplir. Lit aussi les notifications du daemon.
Q1 … Q15Vérifie la mission. Succès → badge + score. Échec → indice niveau 1. Retaper → niveau 2. Retaper → niveau 3 (solution). Les niveaux sont mémorisés dans .hints.
STATUTScore rapide : liste compacte avec OK/-- et total.
⚙ Avant la séance
  • Tester le lancement complet sur un poste élève (JSLinux + import + . ~/tp_infiltration.sh)
  • Vérifier que le daemon démarre : le setup affiche "Daemon actif (PID: …)"
  • Vérifier que stat, ps, find, awk, sh sont disponibles (présents par défaut sur Alpine)
  • Aucun réseau requis — le TP est 100 % local
02 — Daemon de surveillance
Architecture

Le daemon est un script indépendant écrit sur disque (/tmp/mirage_daemon.sh) au moment du setup. Il est lancé avec sh /tmp/mirage_daemon.sh &. Il ne dépend d'aucune fonction du shell parent — c'est la solution au problème classique des sous-shells ash qui n'héritent pas des fonctions sourcées.

Le daemon surveille toutes les 3 secondes et écrit les badges dans ~/.infiltration_mirage/.notifs. La fonction _flush() lit et vide ce fichier au début de chaque commande interactive (Qn, MISSION, STATUT).

03 — Indices progressifs dans Qn

Quand une mission échoue, Qn affiche l'indice du niveau courant et incrémente le compteur dans .hints. Au prochain appel de Qn, le niveau suivant s'affiche.

AppelContenu de l'indice
1er échecRappel de l'objectif + nom et rôle de la commande concernée
2e échecStructure de la commande avec des ??? à compléter
3e échecSolution complète (avec PID réel pour Q14/Q15)
Appels suivantsNiveau 3 ré-affiché (solution toujours disponible)

Les niveaux sont mémorisés par mission dans .hints. Une réinitialisation (re-source du script) remet tout à zéro.

04 — Corrigé des 15 missions
Q1

pwd — Orientation

navigationhistory

Afficher le répertoire courant.

pwd

Pattern history : ^[0-9 ]*pwd\s*$ — pwd seul sur sa ligne.

Q2

ls — Exploration

navigationhistory strict

Lister ~/infiltration_mirage/serveur/.

ls ~/infiltration_mirage/serveur/

Pattern strict : chemin exact requis. ls -la (Q4) ne valide pas Q2.

Q3

ls --help — Documentation

history

Trouver l'option pour les fichiers cachés. Sur JSLinux : man n'est pas installé, utiliser --help.

ls --help   # L'option cherchee est -a (--all)

Pattern : ls\s+--help. L'indice niveau 3 révèle explicitement l'option -a.

Q4

ls -la — Fichiers cachés

navigationhistory strict
ls -la ~/infiltration_mirage/serveur/

Pattern : ls -la ou ls -al uniquement. ls -l -a non accepté.

Q5

cd — Navigation

$PWD + history
cd ~/infiltration_mirage/serveur/
# puis retaper Q5

Double contrôle : $PWD correct OU pattern history. Si l'élève a bougé depuis, la validation history reste acquise.

Q6

cat — Lecture

history
cat ~/infiltration_mirage/message_secret.txt

Pattern : cat\s+.*/message_secret\.txt.

Q7

cat | grep — Introduction du pipe

★ pipe |history strict
cat ~/infiltration_mirage/serveur/logs/access.log | grep Transfert

Pattern : cat.*access\.log.*\|.*grep.*Transfert — pipe ET mot Transfert obligatoires. Point pédagogique central du TP.

Q8

history | grep — Pipe sur l'historique

pipe |history strict
history | grep passwd

Pattern : history\s*\|\s*grep — le pipe est obligatoire, history seul ne valide pas.

Q9

touch — Créer un fichier vide

fichiersdaemon auto
touch ~/infiltration_mirage/serveur/agent.log

Daemon valide dès que agent.log existe. Q9 peut aussi être tapé manuellement.

Q10

find — Recherche récursive

history strict
find ~/infiltration_mirage/ -name "rapport.txt"

Pattern : find.*-name.*rapport — l'option -name est obligatoire.

Q11

mkdir — Créer un dossier

fichiersdaemon auto
mkdir ~/infiltration_mirage/serveur/archive

Daemon valide dès que serveur/archive/ existe.

Q12

chmod 600 — Droits restrictifs

droitsdaemon auto
chmod 600 ~/infiltration_mirage/serveur/exfiltration/rapport.txt
# 600 = rw------- : User(6=rw) Group(0) Others(0)

Daemon vérifie via stat -c "%a". Le fichier existe dès le setup — mission indépendante de Q11.

Q13

chmod u+x — Script exécutable

droitsdaemon auto
chmod u+x ~/infiltration_mirage/serveur/effacer_traces.sh
# u=user  +=ajouter  x=execution

Daemon vérifie via [ -x fichier ].

Q14

ps | grep — Identifier l'espion

processuspipe |history strict
ps | grep espion_mirage   # la ligne 'grep' visible est normale

Pattern : ps\s*\|\s*grep.*(espion|mirage) — pipe obligatoire. L'indice niveau 3 affiche le PID réel en temps réel.

Q15

kill + mv — Neutraliser et camoufler

processusfichiersdaemon auto
kill <PID>   # PID obtenu en Q14 — kill -9 si résistance
mv ~/infiltration_mirage/serveur/exfiltration/rapport_secret.txt \
   ~/infiltration_mirage/serveur/exfiltration/notes_vacances.txt

Daemon valide quand les DEUX conditions sont remplies : espion absent + notes_vacances.txt présent.

05 — Patterns history (stricts)
Principe : mieux vaut ne pas valider que mal valider

Les patterns sont conçus pour ne déclencher que si la bonne commande a été tapée. En cas de doute, l'élève peut retaper la commande puis Qn.

MissionPattern grep -E
Q1 pwd^[0-9 ]*pwd\s*$
Q2 ls^[0-9 ]*ls\s+~/infiltration_mirage/serveur/\s*$
Q3 ls --helpls\s+--help
Q4 ls -lals\s+-la|ls\s+-al
Q5 cd^[0-9 ]*cd\s+.*/infiltration_mirage/serveur\s*$
Q6 catcat\s+.*/message_secret\.txt
Q7 cat|grepcat.*access\.log.*\|.*grep.*Transfert
Q8 history|grephistory\s*\|\s*grep
Q10 findfind.*-name.*rapport|find.*rapport\.txt
Q14 ps|grepps\s*\|\s*grep.*(espion|mirage)
06 — Conseils pédagogiques
Déroulement recommandé
  • 5 min — introduction narrative + rappel légal + distribution fiche élève
  • 10 min — import JSLinux + lancement (Q1/Q2 se valident en quelques secondes)
  • 65–75 min — travail autonome. Encourager les élèves à retaper Qn plutôt que de demander de l'aide immédiatement
  • 15 min — correction collective sur le pipe (Q7), les droits octaux (Q12), kill (Q15)
Points d'attention par mission
  • Q3 ls --help : expliquer que man n'existe pas sur Alpine JSLinux — --help est l'alternative universelle.
  • Q4 ls -la : le pattern n'accepte que -la ou -al. Si un élève tape ls -l -a, lui expliquer la syntaxe condensée.
  • Q5 cd : erreur classique — taper ls serveur/ au lieu de cd serveur/. Insister sur la différence voir / entrer.
  • Q7 pipe : laisser l'élève lire l'indice niveau 1 avant d'intervenir — il explique le principe du pipe.
  • Q12 chmod : décomposer 600 au tableau si besoin : r=4 w=2 x=1, User|Group|Others.
  • Q14 ps : la ligne contenant grep dans les résultats est normale — grep se voit lui-même dans la liste des processus.
  • Q15 kill : distinguer kill (SIGTERM) et kill -9 (SIGKILL). Bonne opportunité pour parler des signaux Unix.
Réinitialiser le TP 
. ~/tp_infiltration.sh

Arborescence recréée, espion relancé, daemon redémarré, progression et niveaux d'indices réinitialisés.
