Opération MIRAGE PROF v5.1

15 missions indépendantes · Double validation (auto + history) · Agent ECHO hors-ligne · JSLinux Alpine
00 — Présentation

TP de terminal Linux pour Terminale NSI, conçu pour JSLinux Alpine (Linux dans le navigateur). Scénario fictif d'infiltration d'un serveur. 15 missions indépendantes réalisables dans n'importe quel ordre.

Nouveautés v5.1 : double validation (watcher filesystem + détection dans history), patterns history stricts pour éviter les fausses validations, agent ECHO entièrement hors-ligne avec 3 niveaux d'indices et rappel systématique de revenir demander de l'aide.

⚠ Rappel légal intégré dans le script. TP 100 % local, aucun réseau externe.

01 — Installation dans JSLinux

Procédure complète à distribuer aux élèves via la fiche élève :

1. Ouvrir bellard.org/jslinux/vm.html?url=alpine-x86.cfg&mem=192

2. Télécharger tp_infiltration.sh sur la machine locale depuis GitHub

3. Cliquer sur la flèche ↑ sous le terminal JSLinux pour importer le fichier

4. Lancer avec le point (pas source — non reconnu par ash) :

~# . ~/tp_infiltration.sh

⚠ Ctrl+V ne fonctionne pas dans JSLinux. Pour coller : clic droit → Paste.

02 — Système de double validation
Trois mécanismes coexistent :
  • Watcher auto (toutes les 3s) — surveille le filesystem et les processus. Valide Q9, Q11, Q12, Q13, Q15 sans intervention de l'élève, dès que la condition est remplie.
  • Vérificateur manuel (Qn) — l'élève tape Q7 par exemple. Contrôle l'état actuel du système.
  • Détection dans history — lors d'un appel Qn, le vérificateur cherche aussi la bonne commande dans les 50 dernières lignes de l'historique. Permet de valider même si l'élève a changé de répertoire ou refait la commande.
Patterns history — volontairement stricts

Les patterns sont conçus pour ne valider que si la commande exacte a été tapée. En cas de doute, la validation manuelle reste possible. Mieux vaut une non-validation qu'une fausse validation.

MissionPattern recherché dans history
Q1 pwd^[0-9 ]*pwd\s*$ — pwd seul sur sa ligne
Q2 ls^[0-9 ]*ls\s+~/infiltration_mirage/serveur/\s*$ — chemin exact
Q3 manman\s+ls
Q4 ls -lals\s+-la ou ls\s+-al uniquement
Q5 cd^[0-9 ]*cd\s+.*/infiltration_mirage/serveur\s*$
Q6 catcat.*message_secret
Q7 pipecat.*access\.log.*\|.*grep.*Transfert
Q8 historyhistory\s*\|\s*grep — pipe obligatoire
Q10 findfind.*-name.*rapport ou find.*rapport\.txt
Q14 psps\s*\|\s*grep.*(espion|mirage) — pipe obligatoire
03 — Agent ECHO hors-ligne
Fonctionnement

AGENT 7 ouvre l'aide pour la mission 7. L'agent est entièrement scriptés, sans IA, sans réseau. Avant de répondre, il analyse l'état réel du système (droits actuels, fichiers présents, PID de l'espion…) et adapte son message.

Chaque réponse se termine par : "Reviens me voir si tu as besoin de plus d'aide : tape AGENT n."

Les 3 niveaux d'indices
  • Niveau 1 — rappel de l'objectif, nom et rôle de la commande concernée
  • Niveau 2 — structure de la commande avec des ??? à compléter
  • Niveau 3 — solution complète (y compris PID réel de l'espion pour Q14/Q15)

L'élève choisit son niveau. Recommandation pédagogique : laisser les élèves commencer par le niveau 1 avant d'aller chercher le niveau 3.

04 — Corrigé des 15 missions
Q1

Orientation — pwd

navigationhistory

Afficher le répertoire courant.

// corrigé
pwd

Validation : appel Q1 (toujours OK) + pattern ^[0-9 ]*pwd\s*$ dans history. Mise en confiance immédiate.

Q2

Exploration — ls

navigationhistory strict

Lister le contenu de ~/infiltration_mirage/serveur/.

// corrigé
ls ~/infiltration_mirage/serveur/

Pattern history : chemin exact requis. ls -la serveur/ (Q4) ne valide pas Q2.

Q3

Documentation — man ls

manhistory

Consulter man ls pour trouver l'option -a (afficher les fichiers cachés).

// corrigé
man ls   # naviguer : Espace / chercher : /motif / quitter : q
# L'option cherchee est -a (--all)

Question déclarative. Pattern history : man\s+ls.

Q4

Infiltration — ls -la

navigationhistory strict

Lister serveur/ avec tous les détails et les fichiers cachés.

// corrigé
ls -la ~/infiltration_mirage/serveur/

Pattern history : ls -la ou ls -al uniquement. ls -l -a non accepté intentionnellement — l'élève doit apprendre la syntaxe condensée.

Q5

Navigation — cd

navigation$PWD + history

Se déplacer dans ~/infiltration_mirage/serveur/.

// corrigé
cd ~/infiltration_mirage/serveur/
# puis retaper Q5 pour valider

Double contrôle : $PWD (position actuelle) OU pattern history. Si l'élève a bougé depuis, la validation history reste acquise avec un message informatif.

Q6

Lecture — cat

lecturehistory

Afficher le contenu de message_secret.txt.

// corrigé
cat ~/infiltration_mirage/message_secret.txt

Pattern history : cat.*message_secret.

Q7

Pipe | — Analyser le journal

★ pipe |history strict

Filtrer access.log pour n'afficher que les lignes contenant Transfert.

// corrigé
cat ~/infiltration_mirage/serveur/logs/access.log | grep Transfert
// explication affichée à l'élève
cat fichier         -> affiche TOUT
cat fichier | grep X -> affiche seulement les lignes avec X
# | = pipe : sortie de gauche -> entree de droite

Pattern history strict : cat.*access\.log.*\|.*grep.*Transfert — le pipe ET le mot Transfert sont obligatoires. Point pédagogique central du TP.

Q8

Pipe | — Historique

pipe |history strict

Chercher passwd dans l'historique des commandes.

// corrigé
history | grep passwd

Pattern history : history\s*\|\s*grep — le pipe est obligatoire. history seul ne valide pas.

Q9

Touch — Créer un fichier vide

fichiersauto-validé

Créer un fichier vide agent.log dans serveur/.

// corrigé
touch ~/infiltration_mirage/serveur/agent.log

Watcher valide dès que le fichier existe (toutes les 3s). Q9 peut aussi être tapé manuellement.

Q10

Find — Rapport confidentiel

findhistory strict

Trouver tous les fichiers rapport.txt dans l'arborescence.

// corrigé
find ~/infiltration_mirage/ -name rapport.txt

Pattern history : find.*-name.*rapport — l'option -name est obligatoire. find ~/infiltration_mirage/ seul ne valide pas.

Q11

Mkdir — Créer un dossier

fichiersauto-validé

Créer le dossier archive/ dans serveur/.

// corrigé
mkdir ~/infiltration_mirage/serveur/archive

Watcher valide dès que serveur/archive/ existe.

Q12

Droits — chmod 600

droitsauto-validé

Appliquer les droits 600 sur exfiltration/rapport.txt.

// corrigé
chmod 600 ~/infiltration_mirage/serveur/exfiltration/rapport.txt
600 = rw------- : User(6=rw) Group(0=---) Others(0=---)

Le fichier existe dès le départ (setup). Watcher contrôle via stat -c "%a". Mission indépendante — pas besoin d'avoir fait Q11.

Q13

Droits — chmod u+x

droitsauto-validé

Rendre effacer_traces.sh exécutable pour son propriétaire.

// corrigé
chmod u+x ~/infiltration_mirage/serveur/effacer_traces.sh
# u=user  +=ajouter  x=execution

Watcher contrôle via [ -x fichier ].

Q14

Processus — top + ps | grep

processuspipe |history strict

Observer les processus avec top, puis trouver le PID d'espion_mirage avec ps | grep.

// corrigé
top   # observer, quitter avec q
ps | grep espion_mirage   # trouver le PID

Pattern history : ps\s*\|\s*grep.*(espion|mirage) — pipe obligatoire. La ligne grep qui apparaît dans ps est normale, l'élève doit l'ignorer.

Q15

Conclusion — kill + mv

processusfichiersauto-validé

Tuer espion_mirage et renommer rapport_secret.txt en notes_vacances.txt.

// corrigé
kill <PID>   # PID obtenu en Q14 — kill -9 si résistance
mv ~/infiltration_mirage/serveur/exfiltration/rapport_secret.txt \
   ~/infiltration_mirage/serveur/exfiltration/notes_vacances.txt

Watcher valide quand les DEUX conditions sont remplies : espion absent (ps | grep vide) ET notes_vacances.txt présent. Q14 et Q15 ciblent des fichiers différents pour être indépendantes.

05 — Conseils pédagogiques
Déroulement recommandé
  • 5 min — introduction narrative + rappel légal + distribution fiche élève
  • 10 min — import JSLinux + lancement (Q1 et Q2 se valident en quelques secondes — mise en confiance)
  • 60–70 min — travail autonome. Encourager AGENT n avant de demander à l'enseignant
  • 15 min — correction collective sur le pipe (Q7), les droits octaux (Q12), kill (Q15)
Points d'attention par mission
  • Q3 man : certains élèves ne savent pas naviguer. Rappeler /mot pour chercher et q pour quitter.
  • Q4 ls -la : pattern history n'accepte que -la ou -al. Si un élève tape ls -l -a, lui expliquer la syntaxe condensée.
  • Q5 cd : erreur classique — taper ls serveur/ au lieu de cd serveur/. La validation history le détecte si c'est bien cd.
  • Q7 pipe : laisser l'élève lire l'explication intégrée avant d'intervenir. Le vérificateur est conçu pour être auto-suffisant.
  • Q12 chmod : décomposer 600 au tableau si besoin (r=4, w=2, x=1).
  • Q14 ps : expliquer que la ligne contenant grep dans les résultats est normale — c'est grep qui se voit lui-même.
  • Q15 kill : distinguer kill (SIGTERM, poli) et kill -9 (SIGKILL, immédiat). Bonne opportunité pour parler des signaux Unix.
Réinitialiser le TP

En cas de problème ou pour repartir de zéro :

. ~/tp_infiltration.sh

Arborescence recréée, espion relancé, watcher redémarré, progression réinitialisée.
